## 四、核心问题清单

### 4.1 架构问题

1. **启动脚本臃肿** - run.py 372行，初始化逻辑混乱
2. **模块耦合严重** - 智能层模块直接依赖采集层具体实现
3. **配置分散** - 配置文件、代码硬编码、环境变量混用
4. **错误处理不统一** - 部分模块有try-catch，部分没有

### 4.2 数据问题

1. **模拟数据不真实** - 原始随机数，无工业特征
2. **智能层数据积累** - 需要长时间运行才有数据
3. **能源数据喂入不完整** - 只有功率，缺少电量累积
4. **SPC数据缺失** - 没有质量数据来源

### 4.3 接口问题

1. **工业4.0接口空** - 大部分接口返回空数据
2. **设备列表格式不一致** - 有的返回列表，有的返回字典
3. **错误响应不规范** - 部分接口返回500而不是标准错误

### 4.4 前端问题

1. **图表设备列表** - 期望字典格式，API返回列表
2. **工业4.0页面** - 依赖数据积累，初始显示空
3. **Toast通知** - 报警通知方式需要改进

## 六、重构方案

### 6.1 重构原则

1. **模块化** - 每个模块职责单一，接口清晰
2. **可测试** - 每个模块可独立测试
3. **可扩展** - 新功能易于添加
4. **符合SCADA标准** - 遵循工业自动化最佳实践

### 6.2 新架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                    展示层 (Presentation)                      │

│  ┌──────────────────────────────────────────────────┐       │
│  │              REST API + WebSocket                 │       │
│  └──────────────────────────────────────────────────┘       │
├─────────────────────────────────────────────────────────────┤
│                    智能层 (Intelligence)                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐       │
```
