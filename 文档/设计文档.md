# 工业数据采集与监控系统设计文档

## 1. 项目概述

### 1.1 项目背景
随着工业4.0和智能制造的发展，工业数据采集与监控系统（SCADA）在工业生产中扮演着越来越重要的角色。本项目旨在开发一个基于Python的轻量级SCADA系统，实现对工业设备的实时数据采集、存储、可视化和报警管理。

### 1.2 项目目标
- 实现Modbus TCP/RTU协议的数据采集
- 提供SQLite数据库存储历史数据
- 开发基于Flask的Web监控界面
- 实现实时数据可视化（ECharts）
- 建立完善的报警管理机制

### 1.3 技术选型
| 模块 | 技术方案 | 选择理由 |
|------|----------|----------|
| 后端框架 | Flask | 轻量级、易扩展、社区活跃 |
| 数据库 | SQLite | 零配置、适合中小规模数据 |
| 前端框架 | Bootstrap 5 | 响应式设计、组件丰富 |
| 图表库 | ECharts | 功能强大、支持实时更新 |
| 通信协议 | pymodbus | Python标准Modbus库 |
| 实时通信 | Flask-SocketIO | WebSocket支持 |

## 3. 详细设计

### 3.1 数据采集模块

#### 3.1.1 Modbus客户端
- 支持Modbus TCP和RTU协议
- 实现功能码01-06的读写操作
- 支持多种数据类型解码（uint16, int16, float32等）
- 提供连接管理和重连机制

#### 3.1.2 数据采集器
- 定时采集任务调度
- 多设备并发采集
- 数据队列缓冲
- 采集统计和错误处理

#### 3.1.3 设备管理器
- 设备配置加载（YAML格式）
- 设备连接状态管理
- 设备增删改查操作

### 3.2 数据存储模块

#### 3.2.1 数据库设计
```sql
-- 实时数据表
CREATE TABLE realtime_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id TEXT NOT NULL,
    register_name TEXT NOT NULL,
    value REAL,
    unit TEXT,
    timestamp DATETIME NOT NULL
);

-- 历史数据表
CREATE TABLE history_data (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    device_id TEXT NOT NULL,
    register_name TEXT NOT NULL,
    value REAL,
    unit TEXT,
    timestamp DATETIME NOT NULL
);

-- 报警记录表
CREATE TABLE alarm_records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    alarm_id TEXT NOT NULL,
    device_id TEXT NOT NULL,
    register_name TEXT NOT NULL,
    alarm_level TEXT NOT NULL,
    alarm_message TEXT,
    threshold REAL,
    actual_value REAL,
    timestamp DATETIME NOT NULL,
    acknowledged BOOLEAN DEFAULT 0
);
```

#### 3.2.2 数据操作
- 数据插入（实时数据、历史数据、报警记录）
- 数据查询（最新数据、历史数据、报警记录）
- 数据聚合（按时间间隔统计）
- 数据清理（按保留策略删除旧数据）

#### 3.2.3 数据导出
- 支持CSV、Excel、JSON格式
- 支持按设备、时间范围导出
- 支持报警记录导出

### 3.3 Web可视化模块

#### 3.3.1 Flask路由
- 首页（仪表盘）

## 4. 接口设计

### 4.1 REST API接口

#### 设备管理
- `GET /api/devices` - 获取所有设备
- `GET /api/devices/<device_id>` - 获取单个设备
- `POST /api/devices/<device_id>/connect` - 连接设备
- `POST /api/devices/<device_id>/disconnect` - 断开设备

#### 数据查询
- `GET /api/data/realtime` - 获取实时数据
- `GET /api/data/latest/<device_id>` - 获取最新数据
- `GET /api/data/history/<device_id>/<register_name>` - 获取历史数据

#### 报警管理
- `GET /api/alarms` - 获取报警记录
- `GET /api/alarms/active` - 获取活动报警
- `POST /api/alarms/<alarm_id>/acknowledge` - 确认报警

#### 系统信息
- `GET /api/system/status` - 获取系统状态
- `GET /api/system/database` - 获取数据库统计

#### 数据导出
- `POST /api/export/device/<device_id>` - 导出设备数据
- `POST /api/export/alarms` - 导出报警记录

### 4.2 WebSocket事件

#### 客户端事件
- `connect` - 连接服务器
- `disconnect` - 断开连接
- `subscribe` - 订阅设备数据
- `unsubscribe` - 取消订阅

#### 服务端事件
- `connected` - 连接成功
- `data_update` - 数据更新
- `alarm` - 报警通知
- `system_status` - 系统状态

## 5. 部署方案

### 5.1 环境要求
- Python 3.10+
- pip包管理器
- 现代浏览器（Chrome、Firefox、Edge）

### 5.2 安装步骤
```bash
# 1. 克隆项目
git clone <repository_url>
cd industrial_scada

# 2. 安装依赖
pip install -r requirements.txt

# 3. 配置设备
编辑 配置/devices.yaml

# 4. 启动系统
python run.py
```

### 5.3 访问系统
- 本地访问：http://localhost:5000
- 局域网访问：http://<ip>:5000
