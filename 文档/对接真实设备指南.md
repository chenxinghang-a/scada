# 对接真实设备指南

## 概述

本系统支持两种运行模式：
- **模拟模式**（默认）：使用仿真数据，适合开发和演示
- **真实设备模式**：连接实际的Modbus设备，用于生产环境

## 快速切换

### 启动真实设备模式

```bash
# 方式1：命令行参数
python run.py --real

# 方式2：Windows批处理
start_real.bat
```

### 启动模拟模式

```bash
# 方式1：命令行参数（默认）
python run.py

# 方式2：Windows批处理
start_simulation.bat
```

## 数据类型说明

| 数据类型 | 说明 | 寄存器长度 | 示例值 |
|---------|------|-----------|--------|
| `uint16` | 无符号16位整数 | 1 | 0~65535 |
| `int16` | 有符号16位整数 | 1 | -32768~32767 |
| `uint32` | 无符号32位整数 | 2 | 0~4294967295 |
| `int32` | 有符号32位整数 | 2 | -2147483648~2147483647 |
| `float32` | 32位浮点数 | 2 | 3.14 |
| `float64` | 64位浮点数 | 4 | 3.141592653589793 |

## 缩放系数说明

缩放系数用于将原始寄存器值转换为实际物理值：

```
实际值 = 原始值 × scale + offset
```

**示例：**
- 寄存器值：250
- 缩放系数：0.1
- 偏移量：0
- 实际值：250 × 0.1 + 0 = 25.0°C

## 报警配置

### 报警规则示例

```yaml
alarm_rules:
  # 温度报警
  - id: temp_high
    name: "温度过高"
    device_id: temp_sensor_01
    register_name: "temperature"
    condition: "greater_than"
    threshold: 35.0
    level: "critical"
    enabled: true
    delay: 10  # 延迟10秒确认
  
  - id: temp_low
    name: "温度过低"
    device_id: temp_sensor_01
    register_name: "temperature"
    condition: "less_than"
    threshold: 10.0
    level: "warning"
    enabled: true
    delay: 10
  
  # 压力报警
  - id: pressure_high
    name: "压力过高"
    device_id: pressure_sensor_01
    register_name: "pressure"
    condition: "greater_than"
    threshold: 0.8
    level: "critical"
    enabled: true
    delay: 5
```

### 报警条件

| 条件 | 说明 | 示例 |
|------|------|------|
| `greater_than` | 大于 | 温度 > 35 |
| `less_than` | 小于 | 温度 < 10 |
| `equal_to` | 等于 | 状态 == 1 |
| `not_equal_to` | 不等于 | 状态 != 0 |
| `between` | 在范围内 | 10 < 温度 < 35 |
| `outside` | 超出范围 | 温度 < 10 或 温度 > 35 |

### 报警级别

| 级别 | 说明 | 颜色 |
|------|------|------|
| `critical` | 严重报警 | 红色 |
| `warning` | 警告 | 黄色 |
| `info` | 信息 | 蓝色 |

## 故障排查

### 1. 设备连接失败

**检查项：**
- [ ] 设备IP地址是否正确
- [ ] 设备是否开机
- [ ] 网络是否连通（ping测试）
- [ ] 防火墙是否放行502端口
- [ ] 从站地址是否正确

**测试命令：**
```bash
# 测试网络连通性
ping 192.168.1.100

# 测试端口是否开放
telnet 192.168.1.100 502
```

### 2. 数据读取失败

**检查项：**
- [ ] 寄存器地址是否正确
- [ ] 数据类型是否匹配
- [ ] 寄存器长度是否正确
- [ ] 设备是否支持该功能码

### 3. 数据值异常

**检查项：**
- [ ] 缩放系数是否正确
- [ ] 偏移量是否正确
- [ ] 数据类型是否匹配（有符号/无符号）

### 4. 报警不触发

**检查项：**
- [ ] 报警规则是否启用
- [ ] 阈值设置是否合理
- [ ] 延迟时间是否过长
- [ ] 设备ID和寄存器名称是否匹配

## 最佳实践

## 常见问题

### Q: 如何知道设备的寄存器地址？

A: 查阅设备的Modbus通信手册，或联系设备厂商获取寄存器表。

### Q: 如何测试Modbus通信？

A: 可以使用Modbus Poll等工具先测试设备通信，确认寄存器地址和数据类型。

### Q: 支持哪些Modbus功能码？

A: 系统支持以下功能码：
- 0x01：读线圈状态
- 0x02：读离散输入
- 0x03：读保持寄存器
- 0x04：读输入寄存器
- 0x05：写单个线圈
- 0x06：写单个寄存器
- 0x0F：写多个线圈
- 0x10：写多个寄存器

### Q: 如何添加新的设备类型？

A: 在`devices.yaml`中添加新的设备配置，按照现有格式填写设备信息和寄存器列表。
