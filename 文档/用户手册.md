# 工业数据采集与监控系统用户手册

## 1. 系统简介

本系统是一个基于Python的工业数据采集与监控系统（SCADA），实现对工业设备的实时数据采集、存储、可视化和报警管理。

## 2. 系统要求

- Python 3.10+
- 现代浏览器（Chrome、Firefox、Edge）

## 3. 安装与启动

### 3.1 安装依赖
```bash
pip install -r requirements.txt
```

### 3.2 配置设备
编辑 `配置/devices.yaml`，配置Modbus设备参数。

### 3.3 启动系统
```bash
python run.py
```

### 3.4 访问系统
打开浏览器访问 `http://localhost:5000`

## 5. 设备配置

### 5.1 配置文件格式
```yaml
devices:
  - id: device_id
    name: "设备名称"
    description: "设备描述"
    protocol: modbus_tcp
    host: "192.168.1.100"
    port: 502
    slave_id: 1
    enabled: true
    collection_interval: 5
    
    registers:
      - name: "temperature"
        description: "温度值"
        address: 0
        length: 1
        data_type: float32
        scale: 0.1
        offset: 0
        unit: "°C"
```

### 5.2 参数说明
- `id`: 设备唯一标识
- `name`: 设备显示名称
- `protocol`: 通信协议（modbus_tcp/modbus_rtu）
- `host`: 设备IP地址
- `port`: 设备端口
- `slave_id`: Modbus从站地址
- `collection_interval`: 采集间隔（秒）

### 5.3 寄存器配置
- `name`: 寄存器名称
- `address`: 寄存器地址
- `data_type`: 数据类型（uint16/int16/float32/float64）
- `scale`: 缩放系数
- `offset`: 偏移量
- `unit`: 单位

## 6. 报警配置

### 6.1 配置文件格式
```yaml
alarm_rules:
  - id: temp_high
    name: "温度过高"
    description: "温度超过上限"
    device_id: temp_sensor_01
    register_name: "temperature"
    condition: "greater_than"
    threshold: 50.0
    level: "critical"
    enabled: true
    delay: 10
```

### 6.2 报警条件
- `greater_than`: 大于
- `less_than`: 小于
- `equal_to`: 等于
- `not_equal_to`: 不等于
- `greater_equal`: 大于等于
- `less_equal`: 小于等于

### 6.3 报警级别
- `critical`: 严重报警
- `warning`: 警告报警
- `info`: 信息提示

## 7. 数据导出

### 7.1 导出设备数据
1. 进入历史数据页面
2. 选择设备和参数
3. 设置时间范围
4. 点击导出按钮
5. 选择导出格式（CSV/JSON）

### 7.2 导出报警记录
1. 进入报警管理页面
2. 点击导出按钮
3. 选择导出格式

## 8. 常见问题

### 8.1 系统无法启动
- 检查Python版本
- 检查依赖包是否安装
- 检查端口是否被占用

### 8.2 设备无法连接
- 检查设备IP地址和端口
- 检查网络连接
- 检查设备是否启用

### 8.3 数据不更新
- 检查设备连接状态
- 检查采集间隔设置
- 检查日志文件
