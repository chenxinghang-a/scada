## 二、采集层（核心重点）

### 2.1 设备管理器 — `采集层/device_manager.py`

**核心类**：`DeviceManager`

**设计思想**：工厂里有不同品牌、不同协议的设备，设备管理器用**协议工厂模式**统一管理，一个类管所有协议设备。

**关键代码**：
```python
def _create_client(config, simulation_mode):
    """根据protocol字段自动创建对应协议客户端"""
    protocol = config.get('protocol', 'modbus_tcp')
    
    if protocol in ('modbus_tcp', 'modbus_rtu'):
        if simulation_mode:
            return SimulatedModbusClient(config)
        else:
            return ModbusClient(config)
    elif protocol == 'opcua':
        return OPCUAClient(config)
    elif protocol == 'mqtt':
        return MQTTClient(...)
    elif protocol == 'rest':
        return RESTDeviceClient(config)
```

**答辩话术**：设备管理器采用了工厂模式，根据配置文件中的protocol字段自动创建对应的协议客户端。新增设备只需在YAML配置文件中添加一条记录，无需修改任何代码，实现"插上就能用"的黑灯工厂理念。

**技术亮点**：
- 配置热加载：运行时修改配置文件，调用`reload_config()`无需重启
- 模拟模式：无真实设备时自动切换模拟数据，方便开发和演示
- 设备状态管理：统一的连接/断开/状态查询接口

### 2.2 Modbus客户端 — `采集层/modbus_client.py`

**核心类**：`ModbusClient`

**设计思想**：封装pymodbus库，支持Modbus TCP（以太网）和Modbus RTU（串口）两种协议，统一读写接口。

**关键代码**：
```python
class ModbusClient:
    def __init__(self, config):
        self.protocol = config.get('protocol', 'modbus_tcp')
        if self.protocol == 'modbus_tcp':
            self.client = ModbusTcpClient(host=self.host, port=self.port)
        else:  # modbus_rtu
            self.client = ModbusSerialClient(
                port=self.serial_port, baudrate=self.baudrate,
                parity=self.parity, stopbits=self.stopbits
            )
    
    def connect(self) -> bool:
        """建立连接，带重试机制"""
        self.client = ModbusTcpClient(host=self.host, port=self.port, timeout=10)
        result = self.client.connect()
        self.connected = result
        return result
```

**答辩话术**：Modbus是工业领域最常用的通信协议。我们封装了pymodbus库，支持TCP和RTU两种模式。TCP走以太网，适用于现代设备；RTU走串口，兼容老设备。通过配置文件切换协议，不需要改代码。

**技术亮点**：
- 双协议支持：TCP（以太网）/ RTU（RS485串口）
- 自动重连：连接断开后自动尝试恢复
- 统计信息：记录成功/失败次数、最后读取时间等
- 寄存器解析：支持float32/float64/uint16/int16等数据类型

### 2.3 OPC UA客户端 — `采集层/opcua_client.py`（黑灯工厂核心）

**核心类**：`OPCUAClient`

**设计思想**：OPC UA是工业4.0标准协议，面向对象、内置安全，是现代PLC的标配接口。客户端在独立线程中运行异步事件循环，通过订阅机制实现数据变更实时推送。

**关键代码**：
```python
class OPCUAClient:
    def connect(self) -> bool:
        """在独立线程中启动异步事件循环"""
        self._thread = threading.Thread(target=self._run_loop, daemon=True)
        self._thread.start()
        return self.connected
    
    async def _connect_and_subscribe(self):
        """建立连接并订阅所有配置的节点"""
        self.client = Client(url=self.endpoint)
        if self.username:
            self.client.set_user(self.username)
            self.client.set_password(self.password)
        
        await self.client.connect()
        self.subscription = await self.client.create_subscription(500, self)
        
        for node_cfg in self.node_configs:
            node = self.client.get_node(node_cfg['node_id'])
            await self.subscription.subscribe_data_change(node)
    
    def datachange_notification(self, node, val, data):
        """数据变更回调 — asyncua框架自动调用"""
        self.latest_data[node_name] = {
            'value': val, 'unit': unit, 'timestamp': datetime.now()
        }
        for callback in self._data_callbacks:
            callback(self.device_id, node_name, val, unit)
```

**答辩话术**：OPC UA是面向工业4.0的标准协议，相比Modbus，它有三个核心优势：第一，面向对象的数据模型，设备数据以节点树形式组织，语义清晰；第二，内置安全认证和加密通信；第三，支持订阅机制，数据变更时服务器主动推送给客户端，不需要轮询。我们用asyncio异步框架实现，在独立线程运行事件循环，避免阻塞主线程。

**技术亮点**：
- 异步架构：asyncio + 独立线程，非阻塞
- 订阅推送：500ms检测间隔，数据变更实时通知
- 安全认证：支持Basic256Sha256加密
- 协议桥接：`OPCUABridge`类将OPC UA数据转为统一格式

### 2.4 MQTT客户端 — `采集层/mqtt_client.py`

**核心类**：`MQTTClient`

**设计思想**：MQTT是物联网消息协议，采用发布/订阅模式。客户端订阅工厂内IoT传感器的主题，实时接收数据。

**关键代码**：
```python
class MQTTClient:
    def __init__(self, broker_host, broker_port, ...):
        self.client = mqtt.Client(client_id=self.client_id, protocol=mqtt.MQTTv311)
        self.client.on_connect = self._on_connect
        self.client.on_message = self._on_message
    
    def _on_message(self, client, userdata, msg):
        """收到消息回调"""
        data = json.loads(msg.payload.decode())
        # 按json_path提取值
        value = self._extract_value(data, topic_config.get('json_path', ''))
        # 触发数据回调
        for callback in self._data_callbacks:
            callback(self.device_id, topic_name, value, unit)
```

**答辩话术**：MQTT是物联网领域最常用的消息协议，采用发布/订阅模式，非常适合无线传感器场景。我们的MQTT客户端支持订阅多个主题，自动解析JSON数据，通过回调机制将数据传递给上层。MQTT的QoS机制保证消息可靠传输。

### 2.5 REST HTTP客户端 — `采集层/rest_client.py`

**核心类**：`RESTDeviceClient`

**设计思想**：现代智能网关和PLC内置Web Server通常提供REST API。通过HTTP请求采集数据，内置轮询线程自动刷新缓存。

**关键代码**：
```python
class RESTDeviceClient:
    def __init__(self, config):
        self.base_url = config.get('base_url')
        self.endpoints = config.get('endpoints', [])
        self.auth_type = config.get('auth_type', 'none')
    
    def read_endpoint(self, endpoint_config):
        """读取单个端点数据"""
        url = self.base_url + endpoint_config['path']
        resp = self._session.request(method=endpoint_config['method'], url=url)
        data = resp.json()
        # 按json_path提取值（支持嵌套路径如 data.sensors[0].value）
        value = self._extract_by_path(data, endpoint_config['json_path'])
        return value
    
    def _poll_loop(self):
        """轮询采集循环（独立线程）"""
        while self._running:
            for ep in self.endpoints:
                value = self.read_endpoint(ep)
                self.latest_data[ep_name] = {'value': value, 'quality': 'good'}
            time.sleep(self.poll_interval)
```

**答辩话术**：REST客户端对接智能网关和PLC的HTTP接口。支持Bearer Token、Basic Auth、API Key三种认证方式。内置JSON路径解析器，可以处理嵌套的API响应。轮询间隔可配置，数据缓存在内存中供上层读取。

### 2.6 数据采集器 — `采集层/data_collector.py`

**核心类**：`DataCollector`

**设计思想**：统一调度所有协议设备的数据采集，区分**轮询型**（Modbus/REST）和**推送型**（OPC UA/MQTT）两种模式。

**关键代码**：

```python
def start(self):

    for device_id, device_config in devices.items():

def _process_data(self):
    """数据处理线程：队列 → 存储 → 报警"""
    while self.running:
        data = self.data_queue.get(timeout=1)
        # 存储到数据库
        self.database.insert_data(...)
        # 检查报警规则
        self.alarm_manager.check_alarm(...)
```

**答辩话术**：数据采集器是整个系统的数据枢纽。它用一个线程安全的队列连接采集层和存储层。对于Modbus和REST设备，采用定时轮询；对于OPC UA和MQTT设备，采用事件驱动——设备数据变更时主动推送过来。所有数据统一进入队列，由独立的处理线程负责存储和报警检查，实现了采集和处理的解耦。

### 3.1 数据库 — `存储层/database.py`

**核心类**：`Database`

**关键设计**：
```python
class Database:
    @contextmanager
    def get_connection(self):
        """上下文管理器：自动提交/回滚/关闭"""
        conn = sqlite3.connect(self.db_path)
        conn.row_factory = sqlite3.Row  # 结果可以按列名访问
        try:
            yield conn
            conn.commit()
        except Exception as e:
            conn.rollback()
            raise e
        finally:
            conn.close()
```

**答辩话术**：存储层使用SQLite嵌入式数据库，零配置、免部署。用Python的上下文管理器管理数据库连接，自动处理提交、回滚和资源释放，避免连接泄漏。数据表分为实时数据表、历史数据表和报警记录表。

```python
class AlarmManager:
    def check_alarm(self, device_id, register_name, value, timestamp):
        """检查单个数据点是否触发报警"""
        for rule_id, rule in self.rules.items():
            if rule['device_id'] == device_id and rule['register_name'] == register_name:
                threshold = rule['threshold']
                level = rule['level']  # warning / danger / critical
                
                if value > threshold:
                    if alarm_state != 'active':

                else:
                    if alarm_state == 'active':
```

```python
    def login(self, username, password, ip_address):
        """登录 → 验证密码 → 检查锁定 → 生成双令牌"""
```

## 八、答辩常见问题预判

### Q: 为什么选择SQLite而不是MySQL/PostgreSQL？
A: 毕设场景数据量不大，SQLite零配置、免部署，一个文件就是整个数据库，方便演示。生产环境可以无缝切换到PostgreSQL，只需要改数据库连接配置。

### Q: OPC UA和Modbus有什么区别？为什么都要支持？
A: Modbus是传统工业协议，简单高效，大量存量设备支持；OPC UA是工业4.0标准，面向对象、有安全认证、支持复杂数据模型。现代工厂是新老设备混用，所以两种都要支持。

### Q: 系统怎么保证数据不丢失？
A: 三层保障：1）数据队列缓冲，采集和存储解耦；2）SQLite的WAL模式保证写入原子性；3）数据归档机制，超过30天的数据自动压缩归档。

### Q: 如何对接真实设备？
A: 只需修改配置文件：填入真实设备的IP地址、端口号、寄存器地址，把simulation_mode改为False，重启系统即可。不需要改任何代码。

### Q: 黑灯工厂体现在哪里？
A: 体现在三点：1）多协议统一接入，新设备插上自动识别；2）全自动数据采集和报警，不需要人工值守；3）远程Web控制，随时随地监控工厂状态。

### Q: 前端怎么支持这么多不同协议？
A: 采用动态表单技术。用户选择协议后，前端JavaScript自动切换表单区域，只显示该协议需要的配置项。后端API也做协议感知验证，不同协议校验不同的必填字段。同时提供预置模板，用户不需要了解底层细节，选模板就能快速配置。

### Q: OPC UA的订阅和MQTT的订阅有什么区别？
A: OPC UA是协议内置的订阅机制，服务端维护订阅关系，数据变更时通过OPC UA协议本身推送，可靠性高，适合工厂内部；MQTT是独立的消息中间件，通过Broker转发，支持跨网络、跨平台，适合IoT场景。两者底层机制不同，但对上层来说都是"数据变更自动推送"，我们的DataCollector用统一的回调接口屏蔽了差异。
