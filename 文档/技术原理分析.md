# 技术原理分析

## 1 Modbus通信协议

### 1.1 Modbus协议概述

Modbus协议是由Modicon公司（现施耐德电气）于1979年提出的一种串行通信协议，是全球第一个真正用于工业现场的总线协议。Modbus协议已经成为工业领域通信协议的业界标准，是工业电子设备之间常用的连接方式。

Modbus协议具有以下特点：
- **开放性**：协议规范公开，无版权要求
- **简单性**：协议帧格式简单，易于理解和实现
- **可靠性**：支持CRC校验，确保数据传输的可靠性
- **灵活性**：支持多种物理层接口（RS-232、RS-485、以太网等）

### 1.2 Modbus通信模型

Modbus协议采用**主从（Master-Slave）通信模型**，通信过程由主站发起，从站响应。

```
┌─────────────┐         ┌─────────────┐
│   主站(Master) │ ──────→ │  从站(Slave)  │
│   (SCADA)    │ ←────── │   (PLC/仪表)  │
└─────────────┘         └─────────────┘
     请求(Request)           响应(Response)
```

**通信规则：**
1. 主站发送请求帧，从站接收并处理
2. 从站返回响应帧，主站接收并解析
3. 同一时刻只能有一个主站发送请求
4. 从站之间不能直接通信

### 1.3 Modbus数据模型

Modbus协议定义了四种基本数据类型：

| 数据类型 | 地址范围 | 访问权限 | 说明 |
|---------|---------|---------|------|
| 线圈（Coil） | 00001-09999 | 读/写 | 1位布尔值，用于控制开关量 |
| 离散输入（Discrete Input） | 10001-19999 | 只读 | 1位布尔值，用于采集开关状态 |
| 输入寄存器（Input Register） | 30001-39999 | 只读 | 16位字，用于采集模拟量 |
| 保持寄存器（Holding Register） | 40001-49999 | 读/写 | 16位字，用于存储参数和数据 |

### 1.4 Modbus功能码

功能码（Function Code）定义了主站请求的操作类型：

| 功能码 | 操作 | 说明 |
|-------|------|------|
| 0x01 | 读线圈状态 | 读取1-2000个连续线圈状态 |
| 0x02 | 读离散输入 | 读取1-2000个连续离散输入 |
| 0x03 | 读保持寄存器 | 读取1-125个连续保持寄存器 |
| 0x04 | 读输入寄存器 | 读取1-125个连续输入寄存器 |
| 0x05 | 写单个线圈 | 写入单个线圈 |
| 0x06 | 写单个寄存器 | 写入单个保持寄存器 |
| 0x0F | 写多个线圈 | 写入多个连续线圈 |
| 0x10 | 写多个寄存器 | 写入多个连续保持寄存器 |

### 1.5 Modbus RTU帧格式

Modbus RTU采用二进制编码，帧格式如下：

```
┌──────────┬──────────┬──────────┬──────────┐
│  从站地址  │  功能码   │   数据    │  CRC校验  │
│  1字节    │  1字节    │  N字节   │  2字节    │
└──────────┴──────────┴──────────┴──────────┘
```

**CRC-16校验算法：**
1. 预置CRC寄存器为0xFFFF
2. 将第一个数据字节与CRC寄存器低字节异或
3. 右移CRC寄存器，检查最低位
4. 如果最低位为1，CRC与0xA001异或
5. 重复步骤2-4，处理所有数据字节
6. 最终CRC值高低字节交换

### 1.6 Modbus TCP帧格式

Modbus TCP在以太网上传输，帧格式如下：

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│  事务标识  │  协议标识  │  长度字段  │  单元标识  │   数据    │
│  2字节    │  2字节    │  2字节    │  1字节    │  N字节   │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

**与RTU的区别：**
- TCP使用502端口
- TCP不需要CRC校验（由TCP协议保证）
- TCP支持多连接并发

## 2 SCADA系统架构

### 2.1 SCADA系统定义

SCADA（Supervisory Control And Data Acquisition）即数据采集与监视控制系统，是以计算机为基础的生产过程控制与调度自动化系统。SCADA系统可以对现场的运行设备进行监视和控制，以实现数据采集、设备控制、测量、参数调节以及各类信号报警等功能。

### 2.2 SCADA系统层次结构

典型的SCADA系统采用分层架构：

```
┌─────────────────────────────────────────────────────────┐
│                    管理层（Management）                    │
│              企业管理系统、决策支持系统                      │
├─────────────────────────────────────────────────────────┤
│                    展示层（Presentation）                   │
│           Web界面、大屏显示、移动终端                        │
├─────────────────────────────────────────────────────────┤
│                    处理层（Processing）                     │
│         数据存储、报警处理、历史记录、报表生成                 │
├─────────────────────────────────────────────────────────┤
│                    通信层（Communication）                  │
│           Modbus TCP/RTU、OPC UA、MQTT                    │
├─────────────────────────────────────────────────────────┤
│                    采集层（Acquisition）                    │
│         PLC、RTU、智能仪表、传感器、执行器                    │
└─────────────────────────────────────────────────────────┘
```

### 2.3 本系统架构设计

本系统采用**四层架构设计**：

```
┌─────────────────────────────────────────────────────────┐
│                      展示层                               │

│    │  仪表盘页面  │  设备管理页面 │  报警管理页面 │          │
│    └─────────────┴─────────────┴─────────────┘          │
│    ┌─────────────┬─────────────┬─────────────┐          │
│    │  历史数据页面 │  系统配置页面 │  数据导出页面 │          │
│    └─────────────┴─────────────┴─────────────┘          │
├─────────────────────────────────────────────────────────┤

│    └─────────────┴─────────────┴─────────────┘          │
└─────────────────────────────────────────────────────────┘
```

**各层职责：**

1. **采集层**：负责与现场设备通信，采集实时数据
   - Modbus客户端：实现Modbus RTU/TCP协议
   - 设备管理器：管理设备连接、状态监控
   - 数据采集器：定时轮询设备，获取数据

2. **存储层**：负责数据持久化存储
   - SQLite数据库：轻量级关系数据库
   - 历史数据存储：存储采集的时序数据
   - 报警记录存储：存储报警事件和处理记录

3. **服务层**：负责业务逻辑处理
   - Flask Web：Python Web框架
   - RESTful API：提供标准化的数据接口
   - WebSocket：实现实时数据推送

4. **展示层**：负责人机交互
   - 仪表盘：显示系统概览和实时数据
   - 设备管理：设备配置和状态监控
   - 报警管理：报警查看和处理

## 3 关键技术

### 3.1 Python Modbus通信

本系统使用pymodbus库实现Modbus通信：

```python
from pymodbus.client import ModbusTcpClient

# 创建Modbus TCP客户端
client = ModbusTcpClient('192.168.1.100', port=502)
client.connect()

# 读取保持寄存器（功能码0x03）
result = client.read_holding_registers(
    address=0,      # 起始地址
    count=2,        # 寄存器数量
    slave=1         # 从站地址
)

# 解析数据
if not result.isError():
    registers = result.registers
    # 将两个16位寄存器组合为32位浮点数
    import struct
    raw = struct.pack('>HH', registers[0], registers[1])
    value = struct.unpack('>f', raw)[0]

client.close()
```

### 3.2 数据类型转换

Modbus寄存器为16位无符号整数，需要根据数据类型进行转换：

| 数据类型 | 转换方法 | 示例代码 |
|---------|---------|---------|
| uint16 | 直接使用 | `value = registers[0]` |
| int16 | 有符号转换 | `value = struct.unpack('>h', struct.pack('>H', registers[0]))[0]` |
| uint32 | 两个寄存器组合 | `value = (registers[0] << 16) + registers[1]` |
| float32 | 两个寄存器转浮点 | `raw = struct.pack('>HH', registers[0], registers[1]); value = struct.unpack('>f', raw)[0]` |

### 3.3 缩放系数处理

工业现场的物理量需要通过缩放系数转换：

```
实际值 = 原始值 × 缩放系数 + 偏移量
```

**示例：**
- 温度传感器原始值：250
- 缩放系数：0.1
- 偏移量：0
- 实际温度：250 × 0.1 + 0 = 25.0°C

### 3.4 报警机制

报警系统采用**阈值比较 + 延迟确认**机制：

```python
def check_alarm(value, rule):
    """检查是否触发报警"""
    if rule['condition'] == 'greater_than':
        triggered = value > rule['threshold']
    elif rule['condition'] == 'less_than':
        triggered = value < rule['threshold']
    
    if triggered:
        # 启动延迟计时器
        if not rule['timer_started']:
            rule['timer_start'] = time.time()
            rule['timer_started'] = True
        
        # 检查延迟时间
        elapsed = time.time() - rule['timer_start']
        if elapsed >= rule['delay']:
            return True  # 确认报警
    
    return False
```

## 4 数据库设计

### 4.1 数据库选型

本系统选用SQLite数据库，原因如下：
- **轻量级**：无需独立服务器进程
- **零配置**：无需安装和配置
- **跨平台**：支持Windows/Linux/macOS
- **性能足够**：满足中小规模数据存储需求

### 4.2 数据表设计

**实时数据表（realtime_data）：**

| 字段名 | 类型 | 说明 |
|-------|------|------|
| id | INTEGER | 主键，自增 |
| device_id | TEXT | 设备ID |
| register_name | TEXT | 寄存器名称 |
| value | REAL | 数据值 |
| unit | TEXT | 单位 |

**历史数据表（history_data）：**

| 字段名 | 类型 | 说明 |
